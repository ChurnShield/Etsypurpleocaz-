## Overview

All data access goes through `SQLiteClient`, a Supabase-compatible query builder. Development uses SQLite; production will use Supabase with the same API.

### What it does

- Provide chainable query builder API (`.table().select().eq().execute()`)
- Singleton pattern via `get_client()` for shared access
- Store workflow registrations, execution records, logs, and proposals

### What it does NOT do

- Handle migrations (schema changes require manual `init_db.py` updates)
- Provide connection pooling (single sqlite3 connection)
- Support transactions beyond single-statement auto-commit

## Schema

**Location**: `scripts/init_db.py`

### workflows

Tracks registered workflows and their run statistics.

| Column | Type | Purpose |
|--------|------|---------|
| id | TEXT PK | Workflow name (e.g., "etsy_analytics") |
| name | TEXT NOT NULL | Display name |
| domain | TEXT | Category (optional) |
| description | TEXT | What this workflow does |
| created_at | DATETIME | First registration time |
| total_runs | INTEGER | Count of all executions |
| successful_runs | INTEGER | Count of successful executions |
| failed_runs | INTEGER | Count of failed executions |
| avg_duration_ms | INTEGER | Average run duration |
| last_run_at | DATETIME | Most recent execution time |

### executions

One row per workflow run.

| Column | Type | Purpose |
|--------|------|---------|
| id | TEXT PK | UUID for this execution |
| workflow_id | TEXT FK | Links to workflows.id |
| started_at | DATETIME | When execution began |
| completed_at | DATETIME | When execution finished |
| status | TEXT | "running", "completed", or "failed" |
| outcome_quality | FLOAT | Quality score (optional) |
| input_summary | TEXT | Brief description of input |
| output_summary | TEXT | Brief description of output |
| error_message | TEXT | Error details if failed |
| metadata | TEXT | JSON string of extra data |

### execution_logs

Detailed event log for every tool call, validation, and phase transition.

| Column | Type | Purpose |
|--------|------|---------|
| id | TEXT PK | UUID for this log entry |
| execution_id | TEXT FK | Links to executions.id |
| workflow_id | TEXT | Which workflow (denormalized for queries) |
| timestamp | DATETIME | When this event occurred |
| phase | TEXT | Current phase name |
| event_type | TEXT | phase_start, phase_end, tool_call, tool_result, validation, error |
| tool_name | TEXT | Which tool (if applicable) |
| validator_name | TEXT | Which validator (if applicable) |
| success | BOOLEAN | Pass/fail for this event |
| duration_ms | INTEGER | Execution time (tool_result only) |
| metadata | TEXT | JSON string of event-specific data |
| error_message | TEXT | Error details (error events only) |

**Indexes**: execution_id, workflow_id, timestamp

### proposals

Improvement proposals generated by SmallBrain.

| Column | Type | Purpose |
|--------|------|---------|
| id | TEXT PK | UUID |
| workflow_id | TEXT FK | Which workflow this applies to |
| generated_at | DATETIME | When SmallBrain created this |
| status | TEXT | "pending", "approved", "rejected", "applied" |
| proposal_type | TEXT | "validator_improvement" or "performance_improvement" |
| title | TEXT | Short description |
| description | TEXT | Full explanation |
| pattern_data | TEXT | JSON: the pattern that triggered this proposal |
| proposed_changes | TEXT | JSON: what should be changed |
| applied_at | DATETIME | When a human applied it (if ever) |
| applied_by | TEXT | Who applied it |

## SQLiteClient Query Builder

**Location**: `lib/common_tools/sqlite_client.py`

### Chainable Methods

```python
db = get_client()  # Singleton

# SELECT
db.table('executions').select('*').eq('workflow_id', 'etsy_analytics').order('started_at', desc=True).limit(10).execute()

# INSERT
db.table('workflows').insert({"id": "my_wf", "name": "My Workflow", ...}).execute()

# UPDATE
db.table('executions').update({"status": "completed"}).eq('id', exec_id).execute()

# DELETE
db.table('proposals').delete().eq('id', proposal_id).execute()
```

### Filter Methods

| Method | SQL equivalent |
|--------|---------------|
| `.eq(col, val)` | `WHERE col = val` |
| `.gt(col, val)` | `WHERE col > val` |
| `.gte(col, val)` | `WHERE col >= val` |
| `.lt(col, val)` | `WHERE col < val` |
| `.lte(col, val)` | `WHERE col <= val` |

Multiple filters combine with AND.

### Return Values

- `SELECT` -> `list[dict]` (each row as a dict)
- `INSERT` -> `[data]` (the inserted dict)
- `UPDATE` -> `[]`
- `DELETE` -> `[]`

## Access Pattern

```python
# ALWAYS use SQLiteClient -- NEVER raw sqlite3
from lib.common_tools.sqlite_client import get_client

db = get_client()  # Uses DATABASE_PATH from config.py

# Or create a specific instance (used in workflow run.py files):
from lib.common_tools.sqlite_client import SQLiteClient
db = SQLiteClient("data/system.db")
```

## Standard Queries

```python
# Count workflow runs
rows = db.table("workflows").select("total_runs").eq("id", workflow_id).execute()
run_count = rows[0]["total_runs"] if rows else 0

# Get recent execution logs
logs = db.table("execution_logs") \
    .select("event_type, tool_name, success, duration_ms") \
    .eq("workflow_id", workflow_id) \
    .order("timestamp", desc=True) \
    .limit(100) \
    .execute()

# Get pending proposals
proposals = db.table("proposals") \
    .select("*") \
    .eq("workflow_id", workflow_id) \
    .eq("status", "pending") \
    .execute()
```

## Data Flow

```
Workflow run.py
    |
    | SQLiteClient.insert() -> executions table (status="running")
    |
    v
ExecutionLogger._buffer (in memory)
    |
    | logger.flush() -> execution_logs table (batch insert)
    |
    v
SmallBrain.analyze()
    |
    | SQLiteClient.select() <- execution_logs (reads patterns)
    | SQLiteClient.insert() -> proposals table (writes proposals)
    |
    v
Human reviews proposals in DB
```

## Database Initialization

```shell
python scripts/init_db.py
```

Creates `data/system.db` with all tables and indexes. Safe to re-run (uses `CREATE TABLE IF NOT EXISTS`). **Warning**: Does not migrate existing data if schema changes.
